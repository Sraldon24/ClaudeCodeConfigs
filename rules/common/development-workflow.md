# Development Workflow
See also `git-workflow.md`.
1. **Research (Mandatory):** `gh search` code/repos. Use `exa-web-search` MCP. Check package registries. Prefer adopting/porting over net-new code.
2. **Plan First:** Use `planner` agent. Generate docs (PRD, architecture). Break down phases, identify risks.
3. **TDD:** Use `tdd-guide`. Write failing tests -> Implement -> Refactor. Aim for >80% coverage.
4. **Code Review:** Use `code-reviewer` agent. Fix CRITICAL/HIGH issues.
5. **Commit:** Use conventional commits.
